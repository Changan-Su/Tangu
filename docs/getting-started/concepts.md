# 核心概念

## Tangu 引擎

所有 AI 能力的执行者。它负责与模型对话、调用工具(读写文件、跑命令、搜索……)、管理 Agent 与会话。桌面端内置了一个引擎实例,开箱即用;高级用户也可以让桌面端连接自己部署的引擎。

## Agent(代理)

一个 Agent = 一个有身份的 AI 伙伴。每个 Agent 拥有:

- **人格设定(SOUL)** — 它是谁、说话什么风格、遵守什么原则;
- **独立记忆** — 与你相处中记下的偏好和事实;
- **资料库(Library)** — 你塞给它的参考资料;
- **自己的模型与工具配置** — 每个 Agent 可以用不同的模型、开放不同的工具。

系统自带默认助手和 [Muse 主动助理](../agents/muse.md),你也可以随意创建新的。详见 [Agent 总览](../agents/overview.md)。

## 会话(Session)

与某个 Agent 的一段对话记录。会话可以随时切换、并行多开,支持从任意一条消息**分支**出新会话。自动化规则也会产生自己的会话,方便回看 AI 每次执行的完整过程。

## Space(空间)

Forsion 的功能以"空间"为单位组织:对话、Amadeus 笔记、日历、收件箱、自动化、编码空间……每个空间是一个独立的功能界面,可以按需启用或关闭,市场里还能安装第三方空间。详见 [Space 总览](../spaces/overview.md)。

## 工作区(Workspace)

- **磁盘上的工作区目录**:你的笔记和文件的家,首次启动时选定,普通文件夹,永远归你。
- **窗口里的工作区布局**:标签页 + 可拖拽分屏。会话、笔记、空间都以标签形式打开,可以左右分栏对照;还支持把内容**拆出独立窗口**,或开成置顶的 **Mini 卡片**小窗。

## 本地模式与云端模式

- **本地模式**(默认):引擎跑在你的电脑上,文件、命令、微信、浏览器自动化这类"碰你机器"的能力全部可用。
- **云端模式**:任务在云端执行(Forsion Web / 移动端即此模式),随处可用,但涉及本机的能力(本地文件、本地命令等)不可用。

## 数据目录

除工作区目录外,Forsion 在用户目录下维护一个 `~/.forsion` 数据目录:账号凭据、配置、Agent 文件、主题、插件都在里面,结构透明。详见[数据与隐私](../reference/data-and-privacy.md)。
