# 数据与隐私

原则只有一条:**本地优先,云端可选**。不开的云功能,数据一个字节都不上传。

## 数据地图

你的所有数据在两个地方,都是普通文件:

| 位置 | 内容 |
|------|------|
| **工作区目录**(首次启动时选定) | 笔记、白板、PDF、多维表、AI 产出的文件 |
| **`~/.forsion` 数据目录** | 配置、账号凭据、Agent(人格 / 记忆 / 日志)、主题、插件、活动记录 |

两个目录拷走 = 完整备份;恢复时放回原位即可。

## 什么会离开你的电脑

只有这些、且只在你使用对应功能时:

- **调用模型** — 对话内容发给你选择的模型服务商(官方云端或你自己配置的服务商),这是 AI 工作的前提;
- **登录 Forsion 账号** — 账号信息与订阅状态;
- **主动开启的云功能** — [云端库 / 目录同步 / 页面共享](../amadeus/cloud-and-sharing.md)、[记忆云同步](../agents/memory.md)所涉及的内容;
- **应用市场** — 安装内容时的下载请求。

## 明确不上传的

- 本地笔记库(未开启同步时);
- [活动记录](../agents/muse.md)(永远只存本机,且默认不采集、需显式授权);
- API Key 与登录凭据(仅存本机,云端调用由应用代理,密钥不落入网页环境)。

## 透明度

- Agent 的记忆是可以打开看的 Markdown;
- 配置是一份 JSON;
- 每个 Agent 干过什么有日志;
- 每条自动化规则跑没跑有记录。

没有"它到底知道我什么"的黑盒。
