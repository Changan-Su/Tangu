# 工具与审批

Tangu 的核心能力是**工具调用**:AI 不只生成文字,还能读写文件、执行命令、搜索网页、管理笔记和日程。权限体系保证它"能干活,但越不了界"。

## 内置工具

大致分几类:

- **文件与命令** — 读写工作区文件、执行终端命令(本地模式);
- **信息获取** — 网页搜索与抓取、读取图片;
- **Forsion 生态** — 操作 Amadeus 笔记、管理日程、发消息到收件箱、创建和配置 Agent;
- **本机集成** — 浏览器自动化、微信收发、桌面操作等(本地模式,部分需在 设置 → 工具 中先行配置)。

## 审批卡:动手前先问你

有风险的操作(执行命令、写文件等)默认**不会直接执行**,而是弹出审批卡,展示它想干什么:

- **批准** — 执行这一次;
- **始终允许** — 本类操作以后免审;
- **拒绝** — 不执行,AI 会另想办法;
- 部分操作还允许你**修改参数后再批准**(比如改掉命令里的某个路径)。

## 计划模式

打开计划模式后,AI 会先**只读**地调研并给出完整计划,经你确认后才开始动手。适合大改动、不熟悉的任务,或你想先看看思路的场合。

## 每个 Agent 的工具开关

在 Agent 配置里可以为它单独设定**工具白名单 / 黑名单**:勾选即允许。给"纯聊天"的角色关掉所有动手能力,给自动化专用的 Agent 只留必要工具,互不越界。

## MCP:接入外部工具

在 设置 → AI → MCP 中可以接入 MCP(Model Context Protocol)服务器,把第三方工具挂给 AI 使用。挂上后与内置工具一样受审批体系约束。

## Hooks(进阶)

设置 → AI → Hooks 允许在工具执行的关键节点挂自定义处理,面向进阶用户。

> 💡 **提示**:自动化规则里的任务没有人守在旁边,审批会一直等不到人。给自动化用的 Agent 配置全自动权限,或只开放无需审批的工具。见[自动化](../spaces/automation.md)。
