# 双链与属性

## 双链 `[[ ]]`

输入 `[[` 立即弹出笔记名补全,选中即建立链接:

- 点击链接直接跳转;
- 链接到**尚不存在**的笔记?照写不误,点击时自动创建——先织网、后填坑的写作方式;
- 同名笔记位于不同文件夹时,按路径就近解析,也可以写 `[[文件夹/笔记名]]` 精确指定;
- 双链同样适用于[白板](whiteboard.md)等非文字内容。

> 💡 **提示**:聊天输入框里也能用 `[[笔记名]]`,把某篇笔记精确递给 AI 参考。

## 子笔记:让笔记有层级

任何笔记都可以拥有**子页面**:在目录树中笔记名旁展开箭头即见。适合"项目笔记下挂会议记录"、"课程笔记下挂每章笔记"这类天然层级。移动、重命名父笔记时,子页面跟随。

## 属性(Properties)

每篇笔记顶部可以挂**属性**——结构化的键值对(基于通行的 YAML frontmatter 格式,与 Obsidian 互通):

- 文本、数字、勾选、日期、列表等类型;
- 在页面顶部的属性面板直接增改;
- 属性存放在文件头部,任何编辑器都能看到。

典型用法:给读书笔记挂"作者 / 评分 / 状态",给项目笔记挂"截止日期 / 负责人"。

> 💡 **提示**:[多维表](databases.md)是属性的放大版——需要按属性筛选、排序、统计一批条目时,用表。
