# 自动化

有些事不值得你每天想起来一次。自动化空间让你把"什么时候 + 让谁 + 干什么"写成规则,之后它自己跑,结果送到你面前。

## 一条规则的三要素

1. **触发** — 什么时候跑:
   - 定时间隔(每 2 小时);
   - 每天定点(每天 9:00);
   - 事件(检测到某情况时);
2. **执行者** — 哪个 [Agent](../agents/overview.md) 来干;
3. **指令** — 干什么,用人话写。

界面提供**线性流构建器**,像搭积木一样把规则拼出来;也可以直接在对话里对 Muse 说,见 [Muse 主动助理](../agents/muse.md)。

## 执行过程全程可查

- 每条规则有自己的**触发记录**:什么时候跑过、结果如何;
- 每次执行都是一个完整会话,点开能看到 AI 当时的每一步——不是黑盒;
- 产出通过通知与[收件箱](inbox.md)送达。

## 等待不花钱

规则守候期间**不消耗模型额度**——只有条件命中、真正开始干活时才唤醒 AI。放一打规则在后台,日常成本为零。

## 权限:无人值守的特殊性

自动化任务跑的时候没人在旁边点"批准"。因此:

- 给自动化规则使用的 Agent 配置**全自动权限**,或只开放无需审批的工具(见[工具与审批](../chat/tools-and-approvals.md));
- 否则任务会停在审批卡上直到超时。

> ⚠️ **注意**:事件类规则自带**冷却时间**,避免"AI 的动作又触发了规则"这类自激循环。

## Agent 日程

除了规则,每个 Agent 还可以有**日程**:一次性或周期性的计划事项。日程可以选择到点自动执行,也可以只做规划提醒;所有日程都会汇入[日历](../amadeus/calendar.md)只读展示。
