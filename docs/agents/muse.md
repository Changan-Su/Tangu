# Muse 主动助理

大多数 AI 等你开口。Muse 反过来:你交代一句"盯着点",它就持续留意,到点、到事、到条件满足时**主动来找你**。

## 盯任务

对 Muse 说人话就能布置监视任务:

- "每天早上 9 点给我今天的日程摘要";
- "我下次打开这个项目的时候提醒我改配置";
- "每隔两小时提醒我起来活动"。

Muse 会把这些登记成规则,后台安静地守着——**等待期间不消耗模型额度**,条件命中才真正唤醒 AI 干活。结果通过通知和[收件箱](../spaces/inbox.md)送达。

规则支持三类触发:

| 类型 | 例子 |
|------|------|
| 定时间隔 | 每 2 小时 |
| 每天定点 | 每天 9:00 |
| 事件 | 检测到某件事发生时 |

## 活动感知(可选授权)

授权后,Muse 能读取**本机活动记录**(你在用什么应用、切换了什么窗口),从而做出情境化的提醒——"你已经连续写了三小时代码了,歇会儿"。

关于这份数据,三件事说清楚:

1. 活动记录**只存在你的电脑上**,不上传;
2. 默认**不开启**,需要你显式授权;
3. 授权范围是按 Agent 控制的——默认只有 Muse,而且随时可以关。

## 日程

Muse(以及任何 Agent)可以维护自己的日程安排,到期自动执行或提醒。日程会汇入 [Amadeus 日历](../amadeus/calendar.md)以只读方式展示,一眼看到"AI 接下来要替你做什么"。

## 与自动化空间的关系

Muse 的盯任务适合口头一句话的轻量委托;需要精细编排(选 Agent、配动作、看执行历史)时,用[自动化空间](../spaces/automation.md)——两者底层同源,规则都在那里可视化管理。
