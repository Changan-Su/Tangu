## Tangu 2.4.0 (2026-07-03)

- **文件预览改在主区标签页打开**——点工作区文件不再从输入框上方弹浮层(浮层暂时停用),改为和聊天/笔记并排的正式标签页:可拖动分屏、可同时开多个、同一文件重复打开自动聚焦已有标签;本机文件的预览标签随布局持久化,重启后自动恢复
- **工作区 .md 直接用笔记编辑器编辑**——预览标签里的本机 Markdown 文件默认进入 Amadeus 所见即所得编辑(与笔记同款),改动自动保存回原文件;frontmatter 原样保留;文件被外部修改时提示「重新加载 / 覆盖写入」,绝不静默覆盖;可切换到源码视图
- **预览不支持的类型询问是否用系统默认应用打开**——新增「用默认应用打开」(工具栏与右键菜单都有),打不开的类型不再只有一句"无法预览"
- **文件面板交互补齐**——右侧栏「文件」树现在支持:单击选中、双击打开;右键菜单(打开/系统默认打开/新建文件/新建文件夹/重命名/复制路径/文件管理器显示/移到回收站);拖动移动到文件夹、Alt+拖拽出到系统、OS 文件拖入复制;行内重命名与新建;智能体跑完自动刷新整棵树
- 预览支持覆盖 PDF/图片/视频/音频/Markdown/HTML(渲染+源码)/代码(含 .py 语法高亮)/JSON/CSV/diff/docx/xlsx/pptx 等主流类型
- **首启引导新增「智能体」步骤**——选完模型后即可认识智能体体系:查看当前名册(含预置助手)、填个名字一句人格就地快速新建;还能一键开启后台智能体 **Historian(史官)**(自动起标题、整理日志与长期记忆,独立/辅助两种模式可选),更多选项仍在 设置 → 后台智能体

## Tangu 2.3.0 (2026-07-03)

- **每个标签页有自己的前进/后退了**——主区左上角的导航箭头改为只作用于当前标签页的浏览历史(会话↔笔记↔微信等切换互不串台),新增快捷键 **Ctrl+{ / Ctrl+}**(mac 为 ⌘⇧[ / ⌘⇧]),原 ⌘/Ctrl+⌥+←/→ 继续可用;快捷键可在设置中重绑定
- **在当前标签页里打开新视图,默认就地切换(Obsidian/浏览器式)**——从启动器、命令、侧栏点开别的主区视图不再另开标签页;想新开用 ＋ 按钮(或 ⌘点击笔记)
- **统一「工作区」侧栏视图**——会话列表、工作区文件、笔记库合并为一个「工作区」视图:随主视图自动切换(对话 → 左栏会话/右栏该会话工作区文件;笔记 → 左栏笔记库/右栏自动定位到笔记所在目录),顶部可手动固定为 会话/文件/笔记,左右侧栏各自独立;旧布局自动迁移
- **统一「大纲」侧栏视图**——对话目录与笔记大纲合并:看对话时是消息目录,看笔记时是标题大纲
- **新建标签页启动器跨空间互通**——主区视图与「最近使用」不再按空间过滤:在任何空间都能直接新建会话、新建笔记、打开最近的某篇笔记/某个会话(Amadeus 项仍随开发者模式开关)
- **聊天输入框支持 `[[` 引用工作区文件**——输入 `[[` 弹出当前工作区文件补全(模糊搜索,回车插入相对路径,智能体可直接读取);仅本机会话
- **侧栏彻底告别左右分屏**——修复焦点在侧栏时 ⌘\ /「向右分屏」命令仍会把侧栏切成左右两半的漏洞(侧栏只允许上下排布)
- **空侧栏占位提示**——侧栏最后一个视图被关掉/拖走后不再整栏消失,显示"这个侧栏是空的,把标签页拖进来试试",可直接拖回
- **Amadeus 编辑器色彩全面跟随主题**——双链 [[wikilink]] 由固定紫色改为跟随主题 accent(含明暗/皮肤切换),数据库弹层阴影、危险按钮、右键菜单红色等硬编码色全部 token 化;编辑器背景改为与聊天视图同一纪律(透出主区纸面,物理上共用同一块底色)
- **编辑器顶栏面包屑改为「悬停展开」**——默认只显示当前笔记标题(居中);鼠标移入顶栏时父级路径从标题左侧逐级淡入展开(标题被平滑推向右、整条保持居中),移出反向收拢;父级弱色、当前标题高亮,悬停任一级可点击定位;父级超过 3 层压缩为「首2 / … / 末1」,当前标题永远完整;展开收起不影响右侧字数与按钮

## Tangu 2.2.0 (2026-07-03)

- **全新「收件箱」空间**——ribbon 新增邮箱式空间:左栏消息列表(发件人/标题/摘要/未读点,支持搜索与 全部/未读/已归档/定时中 筛选,右键管理),主区阅读面板(markdown 正文、归档/删除/标记未读、一键与发件智能体开聊)

- **终端 `tangu` 命令随桌面版自动就位**——安装并首次启动桌面版后,新开终端直接输入 `tangu` 即可使用 CLI(TUI 对话、`tangu login` 等),无需 npm 安装;CLI 直接复用桌面版内置运行时,**桌面版升级即 CLI 升级**,应用被移动/更名后下次启动自动修复命令

## Tangu 2.1.3 (2026-07-02)

- **Amadeus「笔记库」支持嵌套文件夹(Obsidian 式)**——左栏文件视图更名为「笔记库」,文件夹树递归展示子文件夹并按层级缩进(文件夹在前、字母序,空文件夹也可见);右键「新建子文件夹」建出的目录现在真正嵌套呈现,拖拽笔记可落入任意层级,面包屑定位会逐级展开祖先文件夹
- **笔记库显示 vault 里的所有文件**——不再只有 .md 笔记:图片、PDF、音视频、数据库等附件(含 attachments/ 文件夹)都出现在树里(📎 标记);点击用系统默认程序打开,右键可在文件管理器显示或删除,也能拖进任意文件夹;外部往 vault 里放文件,树会自动刷新
- **修复:粘贴/拖入笔记的图片附件一直不显示**——附件确实存进了 vault,但界面的内容安全策略(CSP)从未放行应用自己的附件协议,所有内嵌图片/PDF/音视频预览被静默拦截;现已放行,粘贴图片即时可见
- **笔记编辑器右上角新增 ⋮ 多功能菜单(Obsidian 式)**——导出为 PDF(A4 分页、保留当前排版与图片/公式,导出恒浅色纸面,保存后自动在文件管理器定位)、收藏、在文件管理器中显示、删除笔记
- **新建标签页(＋)全空间统一**——Amadeus 空间的 ＋ 不再直接建笔记,而是与 Tangu 空间一样打开「新建标签页」启动器;启动器按当前空间列出全部可用的主区/侧栏视图(Amadeus 下是 新建笔记/今天 + 笔记库/搜索/标签/大纲/反链/关系图),顶部新增**「最近使用」**——最近打开过的具体笔记、具体会话,一格一卡直达
- **微动效**——笔记库树的箭头旋转、子树展开滑入,启动器卡片错峰入场;均尊重系统「减少动态效果」
- **标签栏 tab 宽度随标题自适应**——短标题的 tab 不再拉到固定宽,按内容收窄(浏览器式);长标题仍在原上限处截断为「…」
- **Historian 周期合一**——不再分「标题每几轮 / 记忆每几轮」两个设置:现在只有一个「每几轮维护」,标题、日志/记忆、辅助讨论全部同一节奏,所见即所得(旧设置自动沿用;内容太少的到期轮仍会跳过,避免琐碎记录)
- **修复:环境检测误报「未安装」**——从 Dock/桌面启动时系统给应用的 PATH 不含 Homebrew 等目录,导致明明装了 node/git/docker 却显示缺失;现已修复检测与安装命令的查找路径
- **「中国大陆」镜像真正生效于引导安装**——环境检测里的安装命令现在也走国内镜像(brew 清华源、pip 清华源、npm npmmirror、docker 官方脚本 Aliyun 源);市场下载的 GitHub 代理改为**多站点自动回退**(单个代理挂了不再整体失败);代码沙箱基础镜像在国内改经 DaoCloud 拉取
- **Linux 带 sudo 的安装命令改为「复制命令」**——需要输密码的命令在应用内执行必然卡死,现在一键复制、去终端粘贴运行

## Tangu 2.1.2 (2026-07-02)

- **修复:回复正文偶发被吞,只剩一句收尾词**——模型「先说一段话、再调工具」时,那段已经流式显示过的正文在落库时会丢失(典型症状:刷新后整条回复只剩「NOTHING」之类的短语);现在各段正文完整保留
- **修复:同一会话前后两轮「换人」**——根因是新会话创建瞬间的一个竞速:历史加载会用服务端尚为空的配置**整体冲掉**刚选好的智能体,第二轮就回落到默认智能体(头像/名字变成 Tangu Arioso)。现已修复竞速(本地配置优先合并 + 创建时序修正),并且**后端会把每轮生效的智能体写穿进会话**——此前被误锁成默认智能体的会话,下一条消息即自动纠正;Historian 辅助讨论也会找对讨论对象
- **修复:后台讨论/群聊/子代理被思考模式模型打断**——DeepSeek 等渠道的思考模式会以「Thinking mode does not support this tool_choice」拒绝部分工具调用参数,导致 Historian 辅助讨论整场失败、群聊/子代理最后一轮偶发报错;已改用兼容方式,投票失败也只会降级不再中断讨论

## Tangu 2.1.0 (2026-07-02)

- **Muse 升级为真正的 Agent**:后台缪斯现在是名册里可见的「Muse」智能体(带「后台」标记),拥有自己的人格(SOUL)、指令与长期记忆——像编辑普通 Agent 一样定制它;它能看到你各个 Agent 的近期动态,还会记住你采纳/忽略过哪些建议,越用越懂你
- **Muse 工作区回归**:「后台智能体」页面重新提供完整的 TODO 工作区——勾选完成、忽略、多选一键注入会话并运行(此前只能查看);注入的任务现在以目标会话自己的 Agent 身份执行
- **Historian 新增「辅助」模式**(设置 → 后台智能体 → Historian → 工作模式):独立模式(默认)下 Historian 自己判断并写入日志/记忆;辅助模式下,到点后它会从当前对话**分支出一场简短的后台讨论**,与该会话的 Agent 商议「有没有值得记的」,由主 Agent 自己决定并亲手写入自己的日志/记忆——你的 Agent 对自己的记忆有最终决定权。标题仍由 Historian 维护;每个会话的首轮始终走独立模式
- 群聊引擎相应增强:讨论可继承来源对话的上下文、可免主持人总结(供后台自动化场景使用)
- **Historian 修复**:开了「共用默认记忆」的 Agent,Historian 此前会把日志/记忆写进错误的文件夹;群聊与外部引擎会话现在也会自动维护标题/日志了
